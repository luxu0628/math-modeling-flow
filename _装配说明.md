# 装配说明：把这条流程真正跑起来

本编排器靠"按 `name` 调用其它 skill"工作，所需 skill 必须安装到 Claude Code 能加载的目录。

## 方式一：一键安装（推荐）

仓库根目录执行：

```bash
# Windows
powershell -ExecutionPolicy Bypass -File install.ps1
# macOS / Linux
bash install.sh
```

脚本会：① 把 `skills/` 里捆绑的 8 个 MIT skill + 编排器本体复制进 `~/.claude/skills/`；
② 从原仓库 `git clone` 拉取 2 个未声明许可的上游（`math-modeling-contest`、
`math-modeling-solver`+`math-modeling-paper`，本仓库不分发其代码）。需要 git 与网络。

## 方式二：手动安装

### 1. skill 清单与来源

| 来源 | skill（`name` 字段） | 角色 |
|---|---|---|
| 本仓库 `skills/`（已捆绑） | `nature-figure`（必装）、`nature-writing`、`nature-polishing`、`nature-academic-search` | 🎨 出图/学术 |
| 本仓库 `skills/`（已捆绑） | `brainstorming`、`systematic-debugging`、`verification-before-completion` | ⚙️ 过程纪律 |
| 本仓库 `skills/`（已捆绑） | `chinese-thesis-workbench` | 🏛️ 治理交付（必装） |
| [xuec699-sudo/math-modeling-skills](https://github.com/xuec699-sudo/math-modeling-skills)（自行 clone，仓库根即 skill） | `math-modeling-contest` | 🦴 脊柱（必装） |
| [Lupynow/math-modeling-skills](https://github.com/Lupynow/math-modeling-skills)（自行 clone，skill 在 `skills/` 下） | `math-modeling-solver`、`math-modeling-paper` | 📚 内容库（必装） |

> 各 skill 的实际 `name` 以其 SKILL.md frontmatter 为准；上游若改版，装前重新核对。

### 2. 安装位置（任选其一）
- **用户级（全局可用）**：`~/.claude/skills/`（Windows 为 `%USERPROFILE%\.claude\skills\`）
- **项目级**：在工作目录放 `.claude/skills/`

把每个 skill 的**整个文件夹**（含 `SKILL.md` 及其 `scripts/`、`static/`、`references/`）放入安装目录，
**文件夹名与 frontmatter 的 `name` 保持一致**；本编排器（根目录 `SKILL.md` + `references/`）放入
`math-modeling-flow/` 文件夹。

⚠️ `math-modeling-solver` 与 `math-modeling-paper` 需作为**同级目录**安装（solver 会跨目录读 paper 的参考文件）。

## 3. 验证
装完后让 Claude 列出可用 skill，确认上表 `name` 都在；再说：

> 用 math-modeling-flow 跑这道题，Manual 模式，先拆题给候选模型等我确认。
