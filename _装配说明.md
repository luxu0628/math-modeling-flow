本编排器靠"按 `name` 调用其它 skill"工作，所以上游 skill 必须先**安装到 Claude Code 能加载的目录**，否则路由会落空。

| [xuec699-sudo/math-modeling-skills](https://github.com/xuec699-sudo/math-modeling-skills) | `math-modeling-contest`（仓库根即 skill） | 🦴 脊柱（必装） |
| [Lupynow/math-modeling-skills](https://github.com/Lupynow/math-modeling-skills) | `math-modeling-solver`、`math-modeling-paper`（在 `skills/` 下） | 📚 内容库（必装） |

> 各 skill 的实际 `name` 以其 SKILL.md frontmatter 为准；上游若改版，装前重新核对。

## 2. 安装位置（任选其一）
- **用户级（全局可用）**：`~/.claude/skills/`（Windows 为 `%USERPROFILE%\.claude\skills\`）
- **项目级**：在工作目录放 `.claude/skills/`

安装方式：clone 上游仓库后，把每个 skill 的**整个文件夹**（含 `SKILL.md`）复制或软链进安装目录，
**文件夹名与 frontmatter 的 `name` 保持一致**。本编排器 `math-modeling-flow/` 也同样放入。

⚠️ 含脚本/引用的 skill（`nature-figure`、`chinese-thesis-workbench`）要连 `scripts/`、`static/`、`references/` 一起带，否则运行时缺文件。
⚠️ `math-modeling-solver` 与 `math-modeling-paper` 需作为**同级目录**安装（solver 会跨目录读 paper 的参考文件）。

## 3. 验证
装完后让 Claude 列出可用 skill，确认上表 `name` 都在；再说：

> 用 math-modeling-flow 跑这道题，Manual 模式，先拆题给候选模型等我确认。

## 4. 可选本地资料区
在工作目录准备 `优秀论文/`（含官方评阅要点）、`算法模型库/`、`论文模板/` 三个文件夹，
对应门会主动引用（详见 SKILL.md"可选本地资料区"节）；没有也能跑。

## 5. 注意
- 只装上表列的 skill，别把 `nature-skills`、`superpowers` 整仓全装（含大量与建模无关的 skill，会稀释触发准确度）。
- 这些是第三方 skill，会随各自仓库更新；赛前重新核对其 `name` 与行为。
- 编排器只负责"调度"，真正的解题质量取决于各专科 skill + **你在 G2/G5 的判断**。
